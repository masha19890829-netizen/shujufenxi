# 信维公式执行版

这份文件是 `system_prompt.md` 的操作化版本，用来约束本地 A 股推荐和复盘工作流。

## 默认结论纪律

- 不再直接说“买这个”，而是先判断是否通过信维公式。
- 六项要素有任意一项为 ❌，默认排除。
- 两项及以上为 ⚠️，只能列为观察，不进入买入候选。
- 只有 S/A 级证据能支撑核心结论；B 级只能作为待验证线索；C 级禁用。

## 候选分层

- **A 档：可进入买入候选**  
  六项全部 ✅，且未来 3 个月至少有一个可验证催化。

- **B 档：观察池**  
  至少 4 项 ✅，其余为 ⚠️，需要等待订单、客户、扣非利润或产能验证。

- **C 档：排除或已错过**  
  出现 ❌，或产业命题已进入从 1 到 N 的充分定价阶段。

## 概率模型默认模板

7 档收益区间：

| 档位 | 收益区间 | 含义 |
|---|---:|---|
| P1 | -50% | 产业命题证伪或重大风险暴露 |
| P2 | -30% | 订单/客户低于预期，估值回落 |
| P3 | -10% | 震荡失败，小幅亏损退出 |
| P4 | +20% | 产业数据兑现一部分 |
| P5 | +50% | 市场开始重估 |
| P6 | +100% | 从 0 到 1 被验证，进入主升 |
| P7 | +200% | 强产业周期叠加稀缺卡位 |

默认计算：

- 胜率 = P4 + P5 + P6 + P7。
- EV = Σ(Pi × Ri)。
- 赔率 = 平均盈利幅度 / 平均亏损幅度。
- 半凯利仓位 = Kelly / 2。
- 单票最终仓位上限：15%。

## 输出时必须避免

- 不用“确定”“必涨”“梭哈”等词。
- 不因一季报差直接判死刑。
- 不把扩张期现金流和短债变化机械理解为坏事。
- 不把股价涨跌当成产业信息本身。
- 不把未验证传闻当作核心证据。

## 后续平台设计方向

本地平台应包含：

- 产业命题数据库：赛道、公司、催化日历、证据来源。
- 信维公式评分器：六项要素逐项评分和证据链接。
- 数据验证工作台：S/A/B/C 信息分级、传闻澄清、订单/送样区分。
- 概率模型计算器：离散概率、EV、赔率、半凯利仓位。
- 监控仪表盘：持有条件、减仓触发、清仓触发、未来 3 个月事件。

推荐技术栈：

- Python
- SQLite
- Streamlit
- 数据接口：优先复用 `a-stock-data`，再按需接入 akshare/tushare。
